Building demo apps on the ONX scaffold

Specs can silently override the locked brand — always re-check tokens + logo

2026-06-03

A spec authored from the scaffold can look complete yet violate the locked brand. The Sales Intelligence spec had recolored the OneNexus brand from teal to navy/gold and swapped the locked <img> logo for a custom SVG. Compare :root in the spec against the scaffold before delivering. The locked targets to restore:

--brand-primary:        #2D7575;
--brand-primary-hover:  #256060;
--brand-primary-soft:   #4EA6A6;
--brand-tint:           #E6F4F4;
--brand-spark:          #B4C850;
--onx-teal: var(--brand-primary);   /* + the var()-based --onx-* chain */
--surface-inverse: #18181B;
--unread: #F97316;
--ring: 0 0 0 3px rgba(45,117,117,0.18);

Then grep for hardcoded brand hexes (#0D1F3C etc.) and re-point them. Keep gold/red/green only as supplementary semantic tokens — never as --brand-*/--onx-*. The OneNexus <img> (base64) lives in the scaffold nav; splice it in to replace any custom logo.

Collapsed sidebar hides .nav-label but NOT .nav-group-label

2026-06-03

When the sidebar collapses to 56px, the scaffold hides .nav-label but leaves section headers (.nav-group-label — "USE CASES", "BANKING", "CAPITAL MARKETS") visible, so they crimp/wrap. Add a collapse rule:

#sidebar.collapsed .nav-group-label {
  opacity: 0; height: 0; padding: 0; margin: 0;
  overflow: hidden; pointer-events: none;
}

(Note: .nav-section-label is auto-hidden by the scaffold, but the use-case nav uses .nav-group-label, which is not.)

Base64 images create 100k+ char lines that block the Read tool

2026-06-03

The scaffold's OneNexus logo and hero images are inline base64, producing single lines of 77k–120k chars. Read fails on these ("exceeds maximum allowed tokens") even with a small limit, because one line blows the budget. Mitigations:

  • Find offenders: awk '{ if (length($0) > 2000) print NR": "length($0) }' file.html
  • Inspect a line's head with awk 'NR==N{print substr($0,1,180)}'
  • Read in ranges that avoid the long lines; use grep -n to locate anchors
  • To splice a base64 asset between files, use a small Python script rather than Edit

window.print() blocks repeated calls inside preview_eval

2026-06-03

The "Print one-pager" / "Print brief" buttons call window.print(), which opens the browser's native print dialog and blocks. Calling it twice in one preview_eval hangs the renderer (30s timeout). When verifying print-adjacent logic (e.g. the Opportunities auto-capture that printRoi() triggers), call the underlying function directly (captureOpportunity(...)) instead of firing printRoi() in a loop.

Doc/version bumping is gated on template changes, not client deliverables

2026-06-03

AGENTS.md requires a CHANGELOG entry + version bump only for template changes. Since the v4.0.0 refactor the version lives once in the header of template/core.css (the single source of truth, ONX Scaffold core.css — vX.X.X); template/layout-shell.html and docs/design-system.html carry a matching <meta name="onx-scaffold-version"> that scripts/pre-commit checks stays in sync. (The old single-file template/onx-app-scaffold.html no longer exists — archived to template/_archive/2026-06-04/.) Building a client app into apps/ (stamped with the existing scaffold version, read from core.css — not bumped) is a normal deliverable output and triggers none of that. Verify with git status: a clean delivery shows only the new apps/*.html as changed. New reusable patterns stay app-specific unless explicitly promoted back into the template (which would then trigger the full doc/version process).
